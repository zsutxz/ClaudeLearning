# Story 2.1: Implement Game State Display

## Description

As a player,
I want to see whose turn it is,
so that I know when it's my turn to play.

## Acceptance Criteria

1. The UI shall clearly indicate which player's turn it is (black or white)
2. The turn indicator shall update immediately after each move
3. The turn indicator shall be visually distinct and easy to understand

## Implementation Plan

### Tasks

- [ ] Design turn indicator UI element
- [ ] Implement turn indicator display logic
- [ ] Ensure immediate update after each move
- [ ] Make turn indicator visually distinct
- [ ] Verify implementation meets all acceptance criteria

### Technical Approach

1. Create UI element for turn indicator
2. Implement logic to update indicator based on game state
3. Ensure real-time updates when moves are made
4. Apply visual styling to make indicator clear and distinct
5. Test with various game scenarios

### Dependencies

- Basic game UI framework
- Turn management system (from Story 1.2)
