# 游戏头脑风暴上下文

这个上下文指南为专注于游戏设计和开发的头脑风暴会议提供游戏特定的考虑因素。

## 会议重点领域

在进行游戏头脑风暴时，考虑探索以下方面：

- **核心玩法循环** - 玩家时时刻刻在做什么
- **玩家幻想** - 游戏满足什么身份/力量幻想？
- **游戏机制** - 定义游戏规则的互动方式
- **游戏动态** - 机制互动产生的涌现行为
- **美学体验** - 激发的情感反应和感受
- **成长系统** - 玩家如何成长和解锁内容
- **挑战与难度** - 如何创建引人入胜的难度曲线
- **社交/多人功能** - 玩家如何相互互动
- **叙事与世界** - 故事、设定和环境叙事
- **艺术方向与手感** - 视觉风格和游戏感觉
- **变现模式** - 商业模式和收入方法（如适用）

## 游戏设计框架

### MDA框架

- **机制（Mechanics）** - 规则和系统（代码中的内容）
- **动态（Dynamics）** - 运行时行为（机制如何互动）
- **美学（Aesthetics）** - 情感反应（玩家的感受）

### 玩家动机（Bartle分类法）

- **成就者（Achievers）** - 目标完成和进度推进
- **探索者（Explorers）** - 发现和理解系统
- **社交者（Socializers）** - 互动和关系建立
- **杀手（Killers）** - 竞争和支配

### 核心体验问题

- 玩家做什么？（动词优先，名词其次）
- 什么让他们感到强大/能力出众/ awesome？
- 核心的紧张感或挑战是什么？
- "再来一回合"的因素是什么？

## 推荐的头脑风暴技巧

### 游戏设计特定技巧

（这些在游戏头脑风暴会议中可作为额外技巧使用）

- **MDA框架探索** - 通过机制-动态-美学进行设计
- **核心循环头脑风暴** - 定义游戏玩法的核心心跳
- **玩家幻想挖掘** - 识别和放大玩家力量幻想
- **类型混搭** - 结合意想不到的类型进行创新
- **动词先于名词** - 专注于行动而非对象
- **失败状态设计** - 从有趣的失败反向工作
- **游戏叙事和谐** - 对齐故事和游戏玩法
- **游戏感觉游乐场** - 纯粹专注于控制的感觉

### 适合游戏的标准技巧

- **SCAMPER方法** - 在现有游戏机制上创新
- **假设情景** - 探索激进的游戏玩法可能性
- **第一性原理思维** - 从零重建游戏概念
- **角色扮演** - 从玩家视角产生想法
- **类比思维** - 从其他游戏/媒体寻找灵感
- **基于约束的创造力** - 围绕限制进行设计
- **形态分析** - 探索机制组合

## 输出指导

有效的游戏头脑风暴会议应该捕获：

1. **核心概念** - 高层次的游戏愿景和吸引点
2. **关键机制** - 主要的游戏玩法动词和互动
3. **玩家体验** - 玩起来的感觉
4. **独特元素** - 让这个游戏特别/不同的地方
5. **设计挑战** - 开发过程中要解决的障碍
6. **原型想法** - 首先要测试什么
7. **参考游戏** - 启发或提供信息的现有游戏
8. **开放问题** - 需要进一步探索的内容

## 与游戏开发工作流程的整合

游戏头脑风暴会议通常会输入到：

- **游戏简介** - 高层次愿景和核心支柱
- **游戏设计文档（GDD）** - 全面设计规范
- **技术设计文档** - 游戏系统架构
- **原型计划** - 为验证概念要构建什么
- **艺术方向文档** - 视觉风格和感觉指南

## 游戏设计的特殊考虑

### 从感觉开始

- 控制应该感觉如何？响应迅速？沉重？飘忽？
- 什么是"游戏感觉" - 那些汁液般的反馈？
- 我们能快速原型化核心互动吗？

### 系统思维

- 机制如何互动？
- 出现什么涌现行为？
- 是否存在主导策略或漏洞利用？

### 为失败而设计

- 玩家如何失败？
- 失败是否有趣和有教育意义？
- 失败的成本是什么？

### 玩家代理权 vs 作者体验

- 玩家在何处有有意义的选择？
- 体验在何处是作者/脚本的？
- 我们如何平衡自由和引导？
