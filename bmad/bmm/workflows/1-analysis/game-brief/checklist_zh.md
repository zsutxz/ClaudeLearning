# 游戏简报验证清单

使用此清单确保您的游戏简报完整且准备就绪，可用于创建游戏设计文档（GDD）。

## 游戏愿景 ✓

- [ ] **核心概念** 清晰简洁（一句话）
- [ ] **电梯演讲** 在2-3句话内吸引读者
- [ ] **愿景声明** 具有启发性但可实现
- [ ] 愿景抓住了您想要创造的情感体验

## 目标市场 ✓

- [ ] **主要受众** 具体明确（不只是"游戏玩家"）
- [ ] 年龄范围和经验水平已定义
- [ ] 游戏会话预期切合实际
- [ ] **市场背景** 展示了机会
- [ ] 竞争格局已被理解
- [ ] 您知道为什么这个受众会关心

## 游戏基础 ✓

- [ ] **核心游戏支柱**（2-4个）明确定义
- [ ] 每个支柱都具体且可衡量
- [ ] **主要机制** 描述玩家实际做什么
- [ ] **玩家体验目标** 将机制与情感联系起来
- [ ] 核心循环清晰且有吸引力

## 范围和约束 ✓

- [ ] **目标平台** 已确定优先级
- [ ] **开发时间线** 切合实际
- [ ] **预算** 与范围匹配
- [ ] **团队资源**（规模、技能）已记录
- [ ] **技术约束** 已识别
- [ ] 范围与团队能力匹配

## 参考框架 ✓

- [ ] **灵感游戏**（3-5个）已列出并附带具体说明
- [ ] 您知道从每个游戏中采用什么和不采用什么
- [ ] **竞争分析** 涵盖直接和间接竞争对手
- [ ] **关键差异化点** 真实且有价值
- [ ] 差异化点具体（不只是"更好"）

## 内容框架 ✓

- [ ] **世界与设定** 已定义
- [ ] **叙事方法** 与游戏类型匹配
- [ ] **内容量** 已估算（大致数量级）
- [ ] 游戏时间预期已设定
- [ ] 重玩方法明确

## 艺术与音频方向 ✓

- [ ] **视觉风格** 已通过参考描述
- [ ] 2D vs 3D 已决定
- [ ] **音频风格** 与游戏情绪匹配
- [ ] **制作方法** 对团队/预算来说切合实际
- [ ] 风格复杂度与能力匹配

## 风险评估 ✓

- [ ] **关键风险** 已诚实识别
- [ ] **技术挑战** 已记录
- [ ] **市场风险** 已考虑
- [ ] **缓解策略** 可操作
- [ ] 需要验证的假设已列出

## 成功标准 ✓

- [ ] **MVP定义** 真正是最小化的
- [ ] MVP可以验证核心游戏假设
- [ ] **成功指标** 具体且可衡量
- [ ] **发布目标** 切合实际
- [ ] 您知道MVP的"完成"是什么样子

## 下一步 ✓

- [ ] **立即行动** 已确定优先级
- [ ] **研究需求** 已识别
- [ ] **开放问题** 已记录
- [ ] 关键路径清晰
- [ ] 阻塞因素已识别

## 整体质量 ✓

- [ ] 简报清晰简洁（3-5页）
- [ ] 各部分内部一致
- [ ] 范围对团队/时间线/预算来说切合实际
- [ ] 愿景有吸引力且可实现
- [ ] 您对构建这个游戏感到兴奋
- [ ] 团队/利益相关者可以理解愿景

## 警示标志 🚩

注意这些警告信号：

- [ ] ❌ 范围对团队/时间线来说过大
- [ ] ❌ 核心循环或支柱不清晰
- [ ] ❌ 目标受众是"所有人"
- [ ] ❌ 差异化点模糊或薄弱
- [ ] ❌ 没有对风险机制的原型计划
- [ ] ❌ 预算/时间线是一厢情愿的想法
- [ ] ❌ 市场饱和，没有明确的定位
- [ ] ❌ MVP实际上不是最小化的

## 准备好进入下一步了吗？

如果您勾选了大部分复选框且没有主要的警示标志：

✅ **准备好继续进行：**

- 原型核心机制
- GDD工作流
- 团队/利益相关者审查
- 市场验证

⚠️ **需要更多工作，如果：**

- 多个部分不完整
- 存在警示标志
- 团队/利益相关者不一致
- 核心概念不清晰

---

_此清单是指南，而非门槛。根据项目需要使用您的判断。_
