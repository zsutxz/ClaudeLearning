# 头脑风暴会话工作流

## 概述

头脑风暴工作流通过使用多样化的创意技术来促进交互式头脑风暴会话。该工作流充当AI引导者，以结构化、充满活力和高度互动的方式引导用户通过各种创意方法来生成和完善创意解决方案。

## 核心特性

- **36种创意技术**：涵盖协作、结构化、创意、深度、戏剧性、狂野和内省方法的全面技术库
- **交互式引导**：AI作为熟练的引导者，采用"是的，而且..."方法论
- **灵活的方法选择**：用户引导、AI推荐、随机或渐进式技术流程
- **上下文感知会话**：通过上下文文档输入支持特定领域的头脑风暴
- **系统化组织**：将想法汇聚为即时机会、未来创新和宏伟目标
- **行动计划制定**：为顶级创意确定优先级，制定具体下一步和时间表
- **会话文档化**：全面的结构化报告，捕捉所有洞察和成果

## 使用方法

### 基本调用

```bash
workflow brainstorming
```

### 使用上下文文档

```bash
# 提供特定领域上下文来引导会话
workflow brainstorming --data /path/to/context.md
```

### 配置

工作流利用 `/bmad/cis/config.yaml` 中的配置：

- **output_folder**：会话结果保存位置
- **user_name**：会话参与者身份标识
- **brain_techniques**：36种创意技术的CSV数据库

## 创意技术库

工作流包含36种技术，分为7个类别：

### 协作技术

- **是的，而且构建**：通过积极补充来建立动力
- **脑力写作循环**：顺序构建的静默创意生成
- **随机刺激**：使用随机催化剂产生意外连接
- **角色扮演**：从多个利益相关者角度生成解决方案

### 结构化方法

- **SCAMPER方法**：通过七个视角（替代/组合/适应/修改/应用/消除/逆向）进行系统化创意
- **六顶思考帽**：通过六个视角（事实/情感/利益/风险/创意/过程）进行探索
- **思维导图**：从中心概念进行视觉分支
- **资源约束**：通过极端限制进行创新

### 创意方法

- **如果情景**：通过质疑约束来探索激进可能性
- **类比思维**：通过领域平行性寻找解决方案
- **逆向反转**：将问题颠倒过来获得新视角
- **第一性原理思维**：剥离假设，从基础重新构建
- **强制关联**：连接不相关概念进行创新
- **时间转换**：探索不同时间段的解决方案
- **隐喻映射**：使用扩展隐喻作为思考工具

### 深度分析

- **五个为什么**：通过因果层钻取到根本原因
- **形态分析**：系统化探索参数组合
- **挑衅技术**：从荒谬的起点提取有用想法
- **假设逆转**：挑战和翻转核心假设
- **问题风暴**：在寻求答案之前先生成问题

### 戏剧性方法

- **时间旅行脱口秀**：采访过去/现在/未来的自己
- **外星人人类学家**：通过完全陌生的眼光审视
- **梦境融合实验室**：从不可能的解决方案开始，逆向工作
- **情感管弦乐**：让不同情绪引导独立会话
- **平行宇宙咖啡厅**：在替代现实规则下探索

### 狂野方法

- **混沌工程**：故意破坏事物以发现稳健解决方案
- **游击园艺想法**：在不合常理的地方种植意外解决方案
- **海盗准则头脑风暴**：从任何地方借鉴有效方法并重新组合
- **僵尸末日规划**：为极端生存场景设计
- **醉鬼历史重述**：用不受抑制的简单性解释

### 内省之乐

- **内在孩童会议**：引导纯真的童年好奇心
- **阴影工作挖掘**：探索您在避免或抗拒的事物
- **价值观考古学**：挖掘驱动决策的深层个人价值观
- **未来自我采访**：从您更智慧的未来自我寻求智慧
- **身体智慧对话**：让身体感觉引导创意生成

## 工作流过程

### 阶段1：会话设置（步骤1）

- 上下文收集（主题、目标、约束）
- 如果提供上下文文档，则进行特定领域指导
- 会话范围定义（广泛探索vs专注创意）

### 阶段2：方法选择（步骤2）

- **用户选择**：浏览并选择特定技术
- **AI推荐**：基于上下文的定制技术建议
- **随机选择**：用于创意突破的惊喜技术
- **渐进流程**：从广泛到专注的多技术之旅

### 阶段3：交互式引导（步骤3）

- 使用问题而非答案的精通引导者方法
- "是的，而且..."构建方法论
- 能量监控和技术切换
- 实时想法捕捉和动力建立
- 数量优于质量焦点（目标：60分钟100个想法）

### 阶段4：汇聚组织（步骤4）

- 回顾和分类所有生成的想法
- 识别技术间的模式和主题
- 分为三个优先级桶用于行动计划

### 阶段5：洞察提取（步骤5）

- 提取跨多种技术的重复主题
- 识别关键认识和意外连接
- 提取更深层次的模式和元洞察

### 阶段6：行动计划（步骤6）

- 为实施确定前3个想法的优先级
- 为每个优先级定义具体下一步
- 确定资源需求和现实时间表

### 阶段7：会话反思（步骤7）

- 分析哪些运作良好，哪些需要进一步探索
- 推荐后续技术和下次会话计划
- 捕捉未来调查的新兴问题

### 阶段8：报告生成（步骤8）

- 编译全面的结构化报告
- 计算生成的总想法数和使用的技巧数
- 格式化所有内容用于分享和未来参考

## 输出

### 生成的文件

- **主要输出**：结构化会话报告保存到 `{output_folder}/brainstorming-session-results-{date}.md`
- **上下文集成**：如果可用，链接到之前的头脑风暴会话

### 输出结构

1. **执行摘要** - 主题、目标、使用技术、生成想法总数、关键主题
2. **技术会话** - 每种技术创意过程的详细记录
3. **想法分类** - 即时机会、未来创新、宏伟目标、洞察
4. **行动计划** - 前3个优先级的理由、步骤、资源、时间表
5. **反思和跟进** - 会话分析、建议、下一步计划

## 要求

- 无特殊软件要求
- 访问CIS模块配置（`/bmad/cis/config.yaml`）
- 在整个交互会话中积极参与和投入
- 可选：专注头脑风暴的领域上下文文档

## 最佳实践

### 开始前

1. **明确意图**：知道您想要广泛探索还是专注问题解决
2. **收集上下文**：准备任何相关背景文档或领域知识
3. **设定时间期望**：为全面会话计划45-90分钟
4. **创造开放环境**：确保创意思维的无干扰空间

### 执行期间

1. **拥抱数量**：生成许多想法，不要自我审查
2. **用"是的，而且"构建**：接受和扩展想法而不是评判
3. **保持好奇**：跟随意外连接和切线
4. **信任过程**：让引导者引导您完成技术转换
5. **记录一切**：记录所有想法，即使是看似愚蠢的
6. **监控能量**：在需要技术变更或休息时进行沟通

### 完成后

1. **24小时内回顾**：在洞察仍然新鲜时重新阅读报告
2. **对快速获胜采取行动**：在一周内实施即时机会
3. **安排跟进**：为有前景的概念规划开发会话
4. **选择性分享**：将相关洞察分发给适当的利益相关者

## 引导原则

AI引导者使用这些核心原则运作：

- **提问，而非告知**：使用问题引出参与者自己的想法
- **构建，而非评判**：使用"是的，而且..."方法论，永远不要"不，但是..."
- **数量优于质量**：在生成阶段追求数量
- **延迟判断**：评估在生成完成后进行
- **保持好奇**：对参与者独特视角表现出真诚兴趣
- **监控能量**：根据参与者参与度调整技术和节奏

## 示例会话流程

### 渐进式技术流程

1. **思维导图**（10分钟）- 建立可能性图景
2. **SCAMPER**（15分钟）- 改进角度的系统化探索
3. **六顶思考帽**（15分钟）- 解决方案的多视角
4. **强制关联**（10分钟）- 意外连接的创意综合

### 能量检查点

- 15-20分钟后："我们应该继续这个技术还是尝试新的？"
- 汇聚阶段前："您准备好开始组织想法，还是继续探索？"
- 行动计划期间："您对最终规划阶段的能量如何？"

## 定制化

要定制此工作流：

1. **添加新技术**：用额外创意方法扩展 `brain-methods.csv`
2. **修改引导风格**：调整 `instructions.md` 中的提示以适应不同能量水平
3. **更新报告结构**：修改 `template.md` 以包含额外分析部分
4. **创建领域变体**：为特定行业开发专门的技术集
